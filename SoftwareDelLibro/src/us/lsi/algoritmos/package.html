Implementaciones de distintos tipos de Algoritmos y los tipos que deben implementar los problemas para ser resueltos por los mismos.

Las clases usadas en el paquete son:

  • La clase {@link us.lsi.ag.AlgoritmoAG AlgoritmoAG} implementa los Algoritmos Genéticos.
  • La clase {@link us.lsi.ag.AlgoritmoAGBinary AlgoritmoAGBinary} implementa los Algoritmos Genéticos con codificación binaria
  • La clase {@link us.lsi.ag.AlgoritmoAGRandomKey AlgoritmoAGRandomKey} implementa los Algoritmos Genéticos con codificación real y clave aleatoria
  • La clase {@link us.lsi.ag.AlgoritmoAGMix AlgoritmoAGMix} implementa los Algoritmos Genéticos un cromosoma binario y otro real
  • La clase {@link us.lsi.bt.AlgoritmoBT AlgoritmoBT} implementa los Algoritmos de Backtracking
  • La clase {@link us.lsi.dyv.AlgoritmoDyV AlgoritmoDyV} implementa los Algoritmos de Divide y Vencerás
  • La clase {@link us.lsi.pd.AlgoritmoPD AlgoritmoPD} implementa los Algoritmos de Programación Dinámica
  • La clase {@link us.lsi.sa.AlgoritmoSA AlgoritmoSA} implementa los Algoritmos de Simulated Annealing
  • La clase {@link us.lsi.voraz.AlgoritmoVZ AlgoritmoVZ} implementa los Algoritmos Voraces
  • La clase {@link us.lsi.algoritmos.Algorimos Algoritmos} implementa una factoría de Algoritmos de los tipos anteriores
  • La clase {@link us.lsi.algoritmos.Soluciones Soluciones} implementa un agregado de soluciones

En el paquete se usan además las clases:

  • La clase {@link us.lsi.flowgraph.FlowAlgorithm FlowAlgorithm} implementa la solución de una red de flujo
  • La clase {@link us.lsi.graphs.AStarAlgorithm AStarAlgorithm} implementa un Algoritmo AStar
  • Se usa la clase SimplexSolver de Apache que implementa el algoritmo del Simplex. Esa clase se adapta y reutiliza en {@link us.lsi.pl.AlgoritmoPL AlgoritmoPL}
  • Se usa la clase LpSolve de que implementa un algoritmo de Programación Lineal Entera. Esa clase se adapta y reutiliza en {@link us.lsi.pl.AlgoritmoPLI AlgoritmoPLI}

El paquete incluye tipos necesarios para usar los algortimos anteriores

  • El tipo {@link us.lsi.bt.EstadoBT EstadoBT} es el tipo del estado para los algortimos de Vuelta Atrás
  • El tipo {@link us.lsi.bt.EstadoBTF EstadoBTF} es el tipo del estado para los algortimos de Vuelta Atrás con Filtro
  • El tipo {@link us.lsi.sa.EstadoSA EstadoSA} es el tipo del estado para los algortimos Simulated Annealing
  • El tipo {@link us.lsi.voraz.EstadoVZ EstadoVZ} es el tipo del estado para los algortimos Voraces
  • El tipo {@link us.lsi.ag.ProblemaAG ProblemaAG} es el tipo de de los problemas que se van a resolver mediante Algoritmos Genéticos
  • El tipo {@link us.lsi.bt.ProblemaBT ProblemaBT} es el tipo de de los problemas que se van a resolver mediante Vuelta Atrás
  • El tipo {@link us.lsi.dyv.ProblemaDyV ProblemaDyV} es el tipo de de los problemas que se van a resolver mediante Divide y Vencerás
  • El tipo {@link us.lsi.pd.ProblemaPD ProblemaPD} es el tipo de de los problemas que se van a resolver mediante Programación Dinámica
  • El tipo {@link us.lsi.pd.ProblemaPDF ProblemaPDF} es el tipo de de los problemas que se van a resolver mediante Programación Dinámica con Filtro
  • El tipo {@link us.lsi.sa.ProblemaSA ProblemaSA} es el tipo de de los problemas que se van a resolver mediante Simulated Annealing
  • El tipo {@link us.lsi.voraz.ProblemaVZ ProblemaVZ} es el tipo de de los problemas que se van a resolver mediante Voraz
